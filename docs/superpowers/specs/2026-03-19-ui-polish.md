# Spec: UI Polish

**Date:** 2026-03-19
**Status:** Draft

---

## Goals

1. Fix alignment issues in sidebar items — icons, text labels, and chevrons must sit on a consistent baseline with no jitter.
2. Raise overall visual quality to a polished, modern Obsidian-like feel.
3. Better typography, tighter spacing rhythm, and more intentional hover/active/focus states.
4. Toolbar improvements — better button grouping and visual hierarchy.
5. CSS and HTML changes only — no backend work.

## Frontend Implementation

### 1. Sidebar item alignment fix

**Root cause (current code):** `.sidebar-item` uses `display: flex; align-items: center; gap: 6px` but the chevron (`▶`, 0.65rem) and folder icon (`📁`, 0.8rem) are different sizes and rendered as Unicode text, causing vertical misalignment depending on the OS font stack. The doc indent (`padding-left: 22px`) is a magic number that doesn't track the chevron column width precisely.

**Fix:**

Introduce a fixed-width left column for the chevron/indent using a CSS grid layout on `.sidebar-item`:

```css
.sidebar-item {
    display: grid;
    grid-template-columns: 16px 16px 1fr;  /* [chevron] [icon] [label] */
    align-items: center;
    column-gap: 4px;
    padding: 4px 8px 4px 6px;
}
```

- Column 1 (16px): chevron for folders, empty for docs.
- Column 2 (16px): emoji icon, sized consistently with `font-size: 13px; line-height: 1`.
- Column 3: label text, `overflow: hidden; text-overflow: ellipsis; white-space: nowrap`.

Docs at root and docs inside folders both use this same grid — the chevron column is just empty for docs. Remove the `.doc-item { padding-left: 22px }` hack entirely.

For nested folders, indent via the **wrapper div's** left padding rather than overriding the item's own padding:

```css
.folder-children { padding-left: 14px; }
```

This means every depth level adds 14px of indent on the container, keeping item internals consistent.

Replace the Unicode chevron (`▶`) with an SVG inline icon (a simple right-pointing triangle path) so it scales predictably across platforms. Rotate it 90deg when open via `transform: rotate(90deg)` on `.folder-item.open .chevron svg`.

### 2. Typography

- Load **Inter** from Google Fonts (already using system-ui; Inter is visually closer to Obsidian's UI).
  ```html
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600&display=swap" rel="stylesheet">
  ```
- Update `--font-ui` to `'Inter', system-ui, -apple-system, sans-serif`.
- Sidebar item font-size: keep `0.875rem` (14px). Line-height: `1.4`.
- Toolbar title: bump to `0.925rem`, `font-weight: 600`.
- Muted text (empty state, version meta): `font-size: 0.8rem`.

```css
:root {

    --accent: #7b5ea7;              /* unchanged */
    --accent-light: #ede9f8;        /* new: used for active bg tint */

    --border: #e0e0e0;              /* slightly lighter */
```

**Dark mode:**

```css
@media (prefers-color-scheme: dark) {
    :root {

```

**Active item styling:**

```css
.sidebar-item.active {
    background: var(--accent-light);
    color: var(--sidebar-active-text);
    font-weight: 500;
}

.sidebar-item.active .icon { opacity: 1; }
```

**Hover with subtle transition:**

```css
.sidebar-item {
    transition: background 0.1s ease;

```

**Sidebar items — tighten margin:**

```css
.sidebar-item {
```

### 4. Hover states

- Add a left accent bar on the active item:

```css
.sidebar-item.active {
    position: relative;
}
.sidebar-item.active::before {
    content: '';
    position: absolute;
    left: 0;
    top: 20%;
    bottom: 20%;
    width: 2px;
    background: var(--accent);
    border-radius: 1px;
}
```

Because `.sidebar-item` already has `border-radius` and `overflow: hidden` is NOT set on it (only on the sidebar itself), the pseudo-element is visible. The item's left margin of 6px means the bar sits inside the rounded rect cleanly.

- Icon opacity: muted when not hovered, full on hover/active:

```css
.sidebar-item .icon { opacity: 0.65; transition: opacity 0.1s; }
.sidebar-item:hover .icon,
.sidebar-item.active .icon { opacity: 1; }
```

### 5. Sidebar header buttons

Current state: three buttons crammed together with emoji labels, no visual grouping.

**Changes:**

- Replace emoji text in sidebar header buttons with SVG icons (inline, ~14px) plus a tooltip via `title` attribute. Remove visible text from the primary `+ Doc` button; use just the icon and a title.
- Group the two "new" actions (`+ Doc`, `+ Folder`) visually and separate the upload button:

```html
<div id="sidebar-header">
  <div class="btn-group">
    <button class="btn icon-btn" id="btn-new-doc" title="New document">
      <!-- SVG: doc + plus icon -->
    </button>
    <button class="btn icon-btn" id="btn-new-folder" title="New folder">
      <!-- SVG: folder + plus icon -->
    </button>
  </div>
  <button class="btn icon-btn" id="btn-upload-doc" title="Upload HTML file">
    <!-- SVG: upload arrow icon -->
  </button>
  <input type="file" id="file-input" accept=".html,text/html" style="display:none;">
</div>
```

```css
#sidebar-header {
    padding: 8px 10px;
    display: flex;
    align-items: center;
    justify-content: space-between;
    gap: 4px;
    border-bottom: 1px solid var(--border);
    flex-shrink: 0;
}

.btn-group {
    display: flex;
    gap: 2px;
}
```

Sidebar header height decreases from the current implicit height (buttons + 12px padding) to a more compact 38px total.

### 6. Toolbar improvements

Current state: flat row with title and buttons, no visual grouping of related actions.

**Changes:**

- Separate destructive action (delete) with a spacer:

```html
<div id="toolbar-actions" style="display:none; gap:4px; align-items:center;">
    <button class="btn" id="btn-toggle-edit">Edit</button>
    <button class="btn" id="btn-save" style="display:none;">Save</button>
    <div class="toolbar-sep"></div>
    <button class="btn" id="btn-history">History</button>
    <button class="btn" id="btn-share">Share</button>
    <div class="toolbar-sep"></div>
    <button class="btn danger" id="btn-delete">Delete</button>
</div>
```

```css
.toolbar-sep {
    width: 1px;
    height: 18px;
    background: var(--border);
    margin: 0 2px;
    flex-shrink: 0;
}

.btn.danger { color: #c0392b; }
.btn.danger:hover { background: #fde8e8; color: #922b21; }

@media (prefers-color-scheme: dark) {
    .btn.danger:hover { background: #4a1515; color: #f1948a; }
}
```

- Remove emoji from toolbar button labels; use short text only ("Edit", "Save", "History", "Share", "Delete").
- Toolbar title (`#toolbar-title`): left-aligned with a subtle `color: var(--muted)` breadcrumb prefix (e.g. folder name / doc name) — **only if** `state.currentFolderId` is set. This is a JS change: update `toolbarTitle.innerHTML` to include a muted folder prefix span.
