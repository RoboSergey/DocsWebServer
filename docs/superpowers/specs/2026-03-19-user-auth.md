# Spec: User Authentication & Per-User Data Isolation

**Date:** 2026-03-19
**Status:** Approved

---

## Goals

1. Add username/password login with JWT tokens.
2. Scope all documents and folders to the authenticated user — users see only their own data.
3. Provide a simple admin page for creating, deleting, and resetting passwords for users.

---

## Non-Goals

- Self-registration (admin creates all accounts).
- Document sharing between users (existing public/token share links are unaffected).
- OAuth / SSO.
- Per-user storage quotas.

---

## Database Changes

### New Table: `users`

```sql
CREATE TABLE users (
    id          TEXT PRIMARY KEY,           -- UUID4
    username    TEXT UNIQUE NOT NULL,
    password    TEXT NOT NULL,              -- bcrypt hash via passlib
    is_admin    BOOLEAN NOT NULL DEFAULT FALSE,
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Modified Tables

```sql
ALTER TABLE documents ADD COLUMN user_id TEXT NOT NULL REFERENCES users(id);
ALTER TABLE folders   ADD COLUMN user_id TEXT NOT NULL REFERENCES users(id);
```

### Migration Strategy

`create_tables()` in `database.py`:

1. Create `users` table (idempotent via `CREATE TABLE IF NOT EXISTS`).
2. `ALTER TABLE documents ADD COLUMN user_id` — wrapped in try/except (already exists = skip).
3. `ALTER TABLE folders ADD COLUMN user_id` — same.
4. Seed: if `users` table is empty, create `admin` / `admin` (`is_admin=True`) and assign all existing orphan documents/folders to this user via `UPDATE documents SET user_id = <admin_id> WHERE user_id IS NULL`.

## Auth Endpoints (`app/routers/auth.py`)

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/auth/login` | `{ username, password }` → `{ access_token, token_type: "bearer" }` |

- Password verified with `passlib.verify`.
- JWT payload: `{ sub: user_id, is_admin: bool, exp: now + 30d }`.
- Signed with `SECRET_KEY` using HS256.
- `GET /login` served by `pages.py` (new template).

---

## Admin Endpoints (`app/routers/admin.py`)

All require `is_admin=True` in the JWT (enforced by `get_admin_user` dependency).

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/admin/users` | List all users |
| `POST` | `/api/admin/users` | Create user `{ username, password, is_admin }` |
| `DELETE` | `/api/admin/users/{id}` | Delete user + cascade delete all their documents/folders |
| `POST` | `/api/admin/users/{id}/reset-password` | Set new password `{ password }` |

- `GET /admin` served by `pages.py` (new template), redirects to `/login` if not admin.

## Dependencies (`app/dependencies.py`)

### `get_current_user`

FastAPI dependency injected into all protected endpoints:

1. Read `Authorization: Bearer <token>` header. Return `401` if missing.
2. Decode JWT with `SECRET_KEY`. Return `401` if invalid or expired.
3. Load user from DB by `sub`. Return `401` if user not found.
4. Return user object.

### `get_admin_user`

Calls `get_current_user`, then checks `user.is_admin`. Returns `403` if not admin.

---

## API Scoping Changes

Every endpoint that reads or writes documents/folders adds `current_user = Depends(get_current_user)` and passes `user_id=current_user.id` to the service layer.

### `document_service.py`

All functions gain a `user_id: str` parameter:

- `list_documents(db, user_id, ...)` — `WHERE user_id = :user_id`
- `create_document(db, user_id, ...)` — sets `document.user_id`
- `get_document(db, doc_id, user_id)` — `WHERE id = :doc_id AND user_id = :user_id` → 404 if not found
- `save_content`, `upload_content`, `delete_document` — all include user_id ownership check

### `folder_service.py`

Same pattern:

- `get_folder_tree(db, user_id)` — `WHERE user_id = :user_id`
- `create_folder(db, user_id, ...)` — sets `folder.user_id`
- `get_folder(db, folder_id, user_id)` — ownership check
- `delete_folder`, `rename_folder`, `reposition_folder` — all include user_id check

### Preview endpoints (unchanged)

`GET /preview/{id}` and `GET /preview/{id}/raw` remain **public** (no auth required) — they are protected by the existing share token system.

## Frontend Changes (`app/static/js/app.js`)

### Token Storage

```javascript
// Store on login
localStorage.setItem('authToken', token);

// Read
const token = localStorage.getItem('authToken');
```

### Authenticated Fetch Wrapper

Replace all `fetch(url, options)` calls with `apiFetch(url, options)`:

```javascript
function apiFetch(url, options = {}) {
    const token = localStorage.getItem('authToken');
    if (!token) { redirectToLogin(); return Promise.reject(); }
    return fetch(url, {
        ...options,
        headers: {
            ...options.headers,
            'Authorization': `Bearer ${token}`,
        },
    }).then(res => {
        if (res.status === 401) { redirectToLogin(); throw new Error('Unauthorized'); }
        return res;
    });
}

function redirectToLogin() {
    localStorage.removeItem('authToken');
    window.location.href = '/login';
}
```

### Startup Check

On `app.js` load, if no token in localStorage → redirect to `/login` immediately.

### Logout

Add a logout button to the toolbar → clears localStorage token → redirects to `/login`.

## New Templates

### `login.html`

- Extends `base.html`.
- Simple centered form: username + password fields + submit button.
- On submit: `POST /api/auth/login` → store token → redirect to `/`.
- Shows error message on invalid credentials.

### `admin.html`

- Extends `base.html` (requires admin token).
- User list table: username, is_admin, created_at, Delete button, Reset Password button.
- "Add User" form: username, password, is_admin checkbox.
- Calls `/api/admin/users` endpoints.

## Out of Scope

- Password change by the user themselves (only admin reset for now).
- Account registration page.
- Session revocation / token blacklist.
- Remember-me / refresh tokens.
- Per-user settings or preferences.
