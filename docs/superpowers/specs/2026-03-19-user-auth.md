# Spec: User Authentication & Per-User Data Isolation

**Date:** 2026-03-19
**Status:** Approved

---

## Goals

1. Add username/password login with JWT tokens.
2. Scope all documents and folders to the authenticated user — users see only their own data.
3. Provide a simple admin page for creating, deleting, and resetting passwords for users.

---

## Non-Goals

- Self-registration (admin creates all accounts).
- Document sharing between users (existing public/token share links are unaffected).
- OAuth / SSO.
- Per-user storage quotas.
- Token refresh / revocation.

---

## Database Changes

### New Table: `users`

```sql
CREATE TABLE IF NOT EXISTS users (
    id          TEXT PRIMARY KEY,           -- UUID4
    username    TEXT UNIQUE NOT NULL,
    password    TEXT NOT NULL,              -- bcrypt hash via passlib
    is_admin    BOOLEAN NOT NULL DEFAULT FALSE,
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Modified Tables

Add `user_id` as a **nullable** column (SQLite does not allow `NOT NULL` without a `DEFAULT` on `ALTER TABLE`). The column is treated as logically required for all new rows; existing rows are backfilled before any new writes occur.

```sql
ALTER TABLE documents ADD COLUMN user_id TEXT REFERENCES users(id);
ALTER TABLE folders   ADD COLUMN user_id TEXT REFERENCES users(id);
```

Both `ALTER TABLE` statements are wrapped in try/except (column already exists = skip).

### Migration Sequence in `create_tables()`

Execute in this exact order:

1. `CREATE TABLE IF NOT EXISTS users (...)` — idempotent.
2. `ALTER TABLE documents ADD COLUMN user_id TEXT` — try/except.
3. `ALTER TABLE folders   ADD COLUMN user_id TEXT` — try/except.
4. **Seed admin user**: if `SELECT COUNT(*) FROM users` is 0, insert a user with `username="admin"`, `password=bcrypt("admin")`, `is_admin=True`, and record its `id`.
5. **Backfill**: `UPDATE documents SET user_id = <admin_id> WHERE user_id IS NULL` and the same for folders.

Note: SQLite cannot retroactively enforce `NOT NULL` via `ALTER TABLE`. The ORM model declares `nullable=False` for new inserts; all existing rows are covered by the backfill in step 5. No further migration is needed.

### Default Credentials Warning

On startup, if the admin account still has the default password `admin`, print a warning to stdout:

```
WARNING: Default admin password is still in use. Change it via the admin panel.
```

## Auth Endpoints (`app/routers/auth.py`)

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/auth/login` | `{ username, password }` → `{ access_token, token_type: "bearer" }` |

- Password verified with `passlib.verify`.
- JWT payload: `{ sub: user_id, is_admin: bool, exp: now + 30d }`.
- Signed with `SECRET_KEY` using HS256 via `PyJWT`.
- Expired or invalid tokens → `401`.

`GET /login` served by `pages.py` (new `login.html` template). No auth required on this route.

---

## Admin Endpoints (`app/routers/admin.py`)

All require `is_admin=True` (enforced by `get_admin_user` dependency → `403` if not admin).

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/admin/users` | List all users |
| `POST` | `/api/admin/users` | Create user `{ username, password, is_admin }` |
| `DELETE` | `/api/admin/users/{id}` | Hard-delete user and all their data |
| `POST` | `/api/admin/users/{id}/reset-password` | Set new password `{ password }` |

### Admin Delete — Hard Delete

When an admin deletes a user, all their documents are **hard-deleted** (not soft-deleted), along with all their folders. Use a direct `DELETE FROM documents WHERE user_id = :id` followed by `DELETE FROM folders WHERE user_id = :id`. Because `Document.versions` has `cascade="all, delete-orphan"` in the ORM, version rows are deleted automatically when documents are deleted via the ORM. Use `db.execute(delete(Document).where(Document.user_id == user_id))` which triggers ORM cascade, then the same for Folder.

`GET /admin` served by `pages.py`. Client-side: `admin.html` checks for a valid admin token on load; if the token is missing or `is_admin` is false, it redirects to `/login`.

## Dependencies (`app/dependencies.py`)

### `get_current_user`

Reads the token **manually from the `Authorization` header** (no `OAuth2PasswordBearer`):

```python
async def get_current_user(
    authorization: str | None = Header(default=None),
    db: AsyncSession = Depends(get_db),
) -> User:
    if not authorization or not authorization.startswith("Bearer "):
        raise HTTPException(status_code=401, detail="Not authenticated")
    token = authorization.removeprefix("Bearer ")
    try:
        payload = jwt.decode(token, settings.secret_key, algorithms=["HS256"])
    except jwt.ExpiredSignatureError:
        raise HTTPException(status_code=401, detail="Token expired")
    except jwt.InvalidTokenError:
        raise HTTPException(status_code=401, detail="Invalid token")
    user = await db.get(User, payload["sub"])
    if not user:
        raise HTTPException(status_code=401, detail="User not found")
    return user
```

### `get_admin_user`

```python
async def get_admin_user(current_user: User = Depends(get_current_user)) -> User:
    if not current_user.is_admin:
        raise HTTPException(status_code=403, detail="Admin required")
    return current_user
```

## API Scoping Changes

Every endpoint that reads or writes documents/folders adds `current_user: User = Depends(get_current_user)` and passes `user_id=current_user.id` to the service layer.

### `document_service.py`

All functions gain a `user_id: str` parameter:

- `list_documents(db, user_id, ...)` — `WHERE user_id = :user_id`
- `create_document(db, user_id, ...)` — sets `document.user_id`
- `get_document(db, doc_id, user_id: str | None = None)` — if `user_id` is provided, adds `AND user_id = :user_id`; if `None`, no user filter (used by unauthenticated preview endpoints). Returns `404` if not found.
- `save_content`, `upload_content`, `delete_document` — pass `user_id` to `get_document` for ownership check.

### `folder_service.py`

- `get_folder_tree(db, user_id)` — two queries inside this function, both need the filter:
  - `select(Folder).where(Folder.user_id == user_id)`
  - `select(Document).where(Document.is_deleted.is_(False), Document.user_id == user_id)`
- `create_folder(db, user_id, ...)` — sets `folder.user_id`
- `get_folder(db, folder_id: str, user_id: str) -> Folder` — new helper, `SELECT ... WHERE id = :folder_id AND user_id = :user_id`, raises `404` if not found. No equivalent exists today; must be created.
- `delete_folder`, `rename_folder`, `reposition_folder` — replace their inline `select(Folder).where(Folder.id == folder_id)` lookups with a call to `get_folder(db, folder_id, user_id)` for ownership enforcement.

### `version_service.py` — Ownership via Router

Version endpoints do **not** receive `user_id` directly. Instead, the `versions.py` router calls `document_service.get_document(db, doc_id, user_id=current_user.id)` first to verify ownership, then proceeds with version operations. Version service functions remain unchanged.

### Preview endpoints (unchanged)

`GET /preview/{id}` and `GET /preview/{id}/raw` in `pages.py` remain **public** (no `get_current_user` dependency). They call `document_service.get_document(db, doc_id, user_id=None)` — no ownership filter.

### Route protection model

- `/` (index) — **client-side only**: `app.js` checks for token in `localStorage` on startup; if missing, redirects to `/login`. The server always serves the index HTML (it cannot see the JWT since it's in `localStorage`, not a cookie).
- `/admin` — same client-side model: `admin.html` checks token on load.
- `/login` — always public.
- All `/api/*` routes (except `/api/auth/login`) — server-side `get_current_user` dependency enforces `401`.

## Frontend Changes (`app/static/js/app.js`)

### Token Storage & Startup

```javascript
// On app.js load (before any other code runs):
if (!localStorage.getItem('authToken')) {
    window.location.href = '/login';
}
```

### Authenticated Fetch Wrapper

Replace all `fetch(url, options)` calls with `apiFetch(url, options)`:

```javascript
function apiFetch(url, options = {}) {
    const token = localStorage.getItem('authToken');
    return fetch(url, {
        ...options,
        headers: {
            ...options.headers,
            'Authorization': `Bearer ${token}`,
        },
    }).then(res => {
        if (res.status === 401) {
            localStorage.removeItem('authToken');
            window.location.href = '/login';
            throw new Error('Unauthorized');
        }
        return res;
    });
}
```

### Logout

Add a logout button to the sidebar header → clears `authToken` from localStorage → redirects to `/login`.

## Test Changes

### `tests/conftest.py`

Override `get_current_user` with a fake user to avoid needing real JWT tokens in tests:

```python
from app.main import app as fastapi_app
from app.dependencies import get_current_user
from app.models import User

fake_user = User(id="test-user-id", username="testuser", password="x", is_admin=True)

@pytest.fixture(autouse=True)
def override_auth():
    # FastAPI accepts sync lambda overrides even for async dependencies
    fastapi_app.dependency_overrides[get_current_user] = lambda: fake_user
    yield
    fastapi_app.dependency_overrides.pop(get_current_user, None)
```

All existing test cases continue to work without modification — the `fake_user` is injected automatically. Tests that specifically test auth flows (login, invalid token, admin-only endpoints) are added separately.

### New: `tests/test_auth.py`

- `POST /api/auth/login` with valid credentials → 200 + token
- `POST /api/auth/login` with wrong password → 401
- `GET /api/documents` with no token → 401
- `GET /api/documents` with expired token → 401
- `GET /api/admin/users` with non-admin token → 403

### `tests/test_dnd.py`, `test_documents.py`, `test_versions.py`, `test_sharing.py`

No changes needed — the `autouse` `override_auth` fixture in `conftest.py` covers them all.

## Out of Scope

- Password change by the user themselves (admin reset only).
- Account registration page.
- Token revocation / blacklist.
- Remember-me / refresh tokens.
- Per-user settings or preferences.
- HttpOnly cookie auth (client-side localStorage is intentional for this SPA).
