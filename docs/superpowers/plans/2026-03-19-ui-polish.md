# UI Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Polish the Obsidian-style UI — fix sidebar item alignment, upgrade typography and colours, unify the sidebar header and toolbar into one continuous bar, and tighten all interactive states.

**Architecture:** Pure frontend changes (CSS, HTML, JS). No backend work. All changes are in four files: `app/templates/base.html`, `app/templates/index.html`, `app/static/css/style.css`, `app/static/js/app.js`. Tasks are ordered so each produces a working, visually testable result.

**Tech Stack:** Vanilla CSS (custom properties, CSS grid), Vanilla JS, Jinja2 HTML templates, Inter font from Google Fonts CDN.

---

## File Map

| File | What changes |

| `app/static/js/app.js` | SVG chevron in `renderFolderList`; `state.folderNames` map in `loadSidebar`; breadcrumb in `selectDocument`; button label text updates |

```javascript
state.folderNames = {};

        if (f.children && f.children.length) collectNames(f.children);
    }
})(folders);
```
